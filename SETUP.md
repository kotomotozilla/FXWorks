# Fraktalex — система сбора отчётности · установка

Три файла:

| Файл | Где живёт | Назначение |
|---|---|---|
| `Code.gs` | Google Apps Script | бэкенд: хранит данные в Google Sheets, шлёт письма |
| `admin.html` | GitHub Pages | панель администратора |
| `employee.html` | GitHub Pages | страница сотрудника |

«Эксель-таблица на Google Drive» — это и есть Google-таблица (лист **Entries**). Выгрузка в `.xlsx`: в таблице **Файл → Скачать → Microsoft Excel**.

---

## Шаг 1. Бэкенд (Google)

1. Создайте Google-таблицу (Google Sheets) — она станет хранилищем.
2. В таблице: **Расширения → Apps Script**. Удалите содержимое `Code.gs` и вставьте весь код из файла `Code.gs`.
3. В блоке `CONFIG` вверху задайте:
   - `ADMIN_PASSCODE` — длинный случайный PIN администратора (не делитесь им).
   - `EMPLOYEE_BASE_URL` — ссылку на страницу сотрудника (заполните после шага 2, формат: `https://USERNAME.github.io/REPO/employee.html`).
   - `SHEET_ID` — можно оставить пустым, если скрипт открыт из самой таблицы.
4. (Необязательно) запустите функцию `setup` один раз — создаст листы. При первом обращении они создаются и так.
5. **Развернуть → Создать развёртывание**:
   - Тип: **Веб-приложение**
   - «Запуск от имени»: **Я**
   - «У кого есть доступ»: **Все** (*Anyone*)
6. Подтвердите доступ к Gmail/Sheets (Google спросит разрешения — это нужно для писем и записи в таблицу).
7. Скопируйте **URL веб-приложения** (`https://script.google.com/macros/s/.../exec`).

Проверка: откройте этот URL в браузере — должно вернуться `{"ok":true,"service":"fraktalex-reports",...}`.

> При изменении кода нужно **Развернуть → Управление развёртываниями → ✏️ → Новая версия**, иначе правки не применятся.

---

## Шаг 2. Фронтенд (GitHub Pages)

1. В `admin.html` и `employee.html` замените `PASTE_YOUR_WEB_APP_URL_HERE` на URL веб-приложения из шага 1.
2. Залейте оба файла в репозиторий GitHub.
3. **Settings → Pages → Source: Deploy from branch**, ветка `main`, папка `/root`. Сохраните.
4. Через минуту страницы доступны:
   - админ: `https://USERNAME.github.io/REPO/admin.html`
   - сотрудник: `https://USERNAME.github.io/REPO/employee.html`
5. Вернитесь в Apps Script, впишите адрес `employee.html` в `EMPLOYEE_BASE_URL` и перевыпустите развёртывание (новая версия).

---

## Шаг 3. Как пользоваться

**Администратор** (`admin.html`):
1. Вход по PIN.
2. **Проекты** → создать проект: название, заказчик, **валюта (USD или EUR)**, общая сумма, **рейт в час**, даты. Система сразу считает количество часов (сумма ÷ рейт) и показывает его.
3. **Отчёты сотрудников** → назначить отчёт: проект + email сотрудника. Валюта, рейт и сумма подставляются из проекта; рейт/сумму можно переопределить под конкретного человека — часы пересчитаются. Комментарий — что отразить в отчёте.
4. Кнопка **«Письмо»** отправляет сотруднику уведомление со ссылкой.
5. Когда сотрудник отправит данные — статус станет «Отчёт получен», кнопка **«Отчёт»** покажет содержимое. Всё также копится во вкладке **Собранные данные** и в Google-таблице.

**Сотрудник** (`employee.html`):
1. Переходит по ссылке из письма → вводит свой email.
2. Видит список своих отчётов: **на сколько часов нужно отчитаться**, рейт, сумма, даты, заказчик, комментарий.
3. Открывает отчёт, добавляет пункты работ (описание + часы). В шапке live: целевые часы, рейт, сколько уже отчитано (часы и деньги) и остаток.
4. Отправляет. Можно вернуться и отредактировать — отправка перезаписывает прежние данные этого отчёта.

## Что важно знать про безопасность

- **Сотрудник** «аутентифицируется» только по email — это намеренно просто, как в задании. Любой, кто знает чужой email, увидит его отчёты. Данные здесь не критичные (часы/описания), но имейте это в виду. Если нужно строже — добавлю одноразовый токен в ссылку из письма (magic-link), скажите.
- **Админ** защищён PIN-кодом. PIN проверяется сервером при каждом действии. Меняйте его на длинный и не публикуйте `Code.gs` с реальным PIN в открытом репозитории.
- Развёртывание «Все» (*Anyone*) означает доступность endpoint без входа в Google — это нужно, чтобы статические страницы могли обращаться к API. Сами действия защищены: админские — PIN-ом, сотрудника — проверкой email на сервере.

## Если что-то не работает

- **Письма не приходят** → проверьте «Спам»; убедитесь, что при развёртывании выдали разрешение на Gmail. Лимит Gmail — порядка 100 писем/сутки на обычном аккаунте.
- **Ошибка в браузере при запросе (CORS)** → endpoint должен быть открыт «Все», и URL должен заканчиваться на `/exec` (не `/dev`).
- **Правки в коде не видны** → создайте новую версию развёртывания.
- **«Не настроен APPS_SCRIPT_URL»** → не заменили `PASTE_...` в html.

## Модель данных (листы таблицы)

- **Projects**: проект — название, заказчик, валюта, сумма, рейт, часы (рассчитываются), даты.
- **Assignments**: назначение отчёта сотруднику — проект + email + валюта + рейт + сумма + часы + комментарий + статус.
- **Entries**: строки отчётов — по одной на каждый пункт работ (валюта, описание, часы, сумма). Это финальная «эксель-таблица».

> Если вы уже разворачивали раннюю версию и листы созданы со старыми колонками — удалите листы **Projects**, **Assignments**, **Entries** в таблице. Они пересоздадутся с новыми колонками при первом обращении.
